# Changelog

## v4.0.0 — 2026-03-28

### 新機能

- **receiver多言語UI** — i18next 導入とナビバー言語切替を追加し、英語/日本語の表示切替に対応

### 改善

- **RTMP URL連携制御** — URL自動取得トグルを追加し、関連ラベルと内部キー構成を整理
- **receiver構成整理** — `index.html` からスクリプトを分離し、画面構成と保守性を改善
- **Opusエンコード調整項目** — ビットレート・サンプルレートを設定可能に（複雑度は10固定）

### 修正

- **適用遅延とUI状態** — `apply_delay` の適用挙動、プロパティ再描画ループ、RTMP URL自動入力の安定性を修正

### ビルド・配布

- **receiver配布方式変更** — npmビルド成果物ベースへ統一し、配布ZIP同梱内容を更新

## v3.2.0 — 2026-03-28

### 改善

## v3.1.0 — 2026-03-27

### 修正

- **重複フィルター時の挙動** — obs-delay-stream フィルター重複時はクラッシュさせず、警告表示で案内

## v3.0.0 — 2026-03-27

- **サブCHパフォーマー名** — チャンネルごとに名前を設定可能に。レシーバUIにも表示
- **ストリームID自動生成** — stream_id の自動生成とロック編集対応

### 改善

- **WS送信コントロール改善** — 遅延トグルUIの更新
- **レシーバUI更新** — WebSocketサーバーとの連携強化
- **メモ欄の非表示制御** — 空のとき自動で非表示
- **CHバリデーション** — WS接続時のチャンネル範囲チェック追加
- **クリップボードUnicode対応** — コピー時の文字化け修正

### ビルド・配布

- **プラグイン表示名変更** — obs-delay-stream に統一
- **ビルド設定更新** — build.bat の整理、SVG一括処理、Boothリンク追加
- **PDF生成修正** — wkhtmltopdf への切り替え

## v2.1.0 — 2026-03-26

### 改善

- **SyncFlow Step1 失敗チャンネルのリトライ** — 一部チャンネルが計測失敗しても、成功済みの結果を保持したまま失敗分だけ再計測可能に
- **WS チャンネルキャッシュの永続化** — サーバー再起動後も計測結果・適用済み遅延値を保持し、再接続クライアントに自動復元

### ビルド・配布

- ブランディングアセット（ロゴ・favicon・ナビバー SVG）追加、レシーバ UI に反映
- ProgramData / Program Files (legacy) デュアルインストールレイアウト対応
- GitHub Issue テンプレート（バグ報告・機能要望）追加
- リポジトリを burgermate → MZ1987Records に移管

## v2.0.0 — 2026-03-25

### 新機能

- **Opus エンコーディング対応** — WebSocket 配信の音声を Opus で圧縮して送信可能に。帯域を大幅に削減（PCM との切り替えも可能）
- **WASM Opus デコーダ同梱** — ブラウザがネイティブ Opus デコードに対応していない場合でも、WASM フォールバックで再生可能
- **受信ページの DLL 埋め込み** — receiver HTML を WebSocket ポート(19000)から直接配信。別途ファイルを配置する必要がなくなった
- **日本語 UI 対応** — プラグイン設定画面が日本語で表示されるように（en-US / ja-JP）
- **cloudflared 自動ダウンロード** — トンネル起動時に cloudflared.exe が未インストールなら自動取得。手動セットアップ不要に
- **トンネル UX 改善** — トンネル種別やパス設定の UI をわかりやすく整理

### ビルド・配布

- GitHub Actions による自動ビルド・プレリリース作成（タグプッシュ時）
- `build.bat` ワンクリックビルド対応（OBS 自動取得・ビルド・インストールまで一括）
- `build.env` による設定カスタマイズ（OBS ソースパス、インストール先の切替など）

## v1.0.0 — 2026-03-23

### 初期リリース

- 音声遅延処理（ms 単位、ON/OFF 切替）
- サブ CH x10 の WebSocket 配信（ch ごとに個別遅延設定）
- 同期フロー（3 ステップで全パフォーマーの遅延を自動最適化）
- パフォーマー遅延計測（ping/pong RTT）
- RTMP 遅延計測（ハンドシェイク RTT）
- cloudflared トンネル内蔵（IP 隠蔽）
