# Technical Notes

## OBSコールバックサイズとOpusフレームサイズの不一致によるバッファ揺れ

OBSの音声コールバックは1024サンプル単位（48kHz時 ≈ 21.3ms）で呼ばれるが、
Opusエンコーダのフレームサイズは960サンプル（48kHz時 = 20ms）である。

この不一致により、FIFO経由でエンコードする際に約15回に1回、
1回のコールバックから2つのOpusフレームが同時に生成・送信される。

```
コールバック1:  FIFO 0→1024,   エンコード→1パケット (残64)
コールバック2:  FIFO 64→1088,  エンコード→1パケット (残128)
...
コールバック15: FIFO 896→1920, エンコード→2パケット (残0)
```

レシーバー側では、2パケットがほぼ同時に到着するとバッファ表示が一時的に跳ね上がり、
その後徐々にドレインして設定値に戻る。

- 再生バッファを20msに設定した場合、接続直後は平均30ms付近に見えることがある
- しばらく再生を続けると20msに安定する
- 最小バッファは設定値どおり保たれており、実害はない

関連コード: `stream-router.cpp` `encode_opus_packets()` / `receiver/src/audio.ts` `playBuffer()`
