# release-tag スキル

バージョン番号の更新、CHANGELOG 追記、コミット、タグ作成、プッシュまでを一括で行う。

## 引数

`$ARGUMENTS` の解釈:

| 入力例 | 動作 |
|--------|------|
| `2.1.0` / `v2.1.0` | そのバージョン番号を使う（`v` は除去） |
| `major` / `minor` / `patch` | 現在バージョンに対して指定レベルをバンプ |
| （空） | コミット内容から自動判定（下記ルール） |

### バージョン形式の制約

このスキルは **正式リリース (`X.Y.Z`) のみ** を扱う。

- 指定されたバージョン番号（`v` 除去後）が `X.Y.Z`（各要素は非負整数）に合致しない場合はエラーとして中断する。
- ハイフン等で始まるサフィックスが付いている形式（例: `1.0.0-rc.1`, `2.0.0-beta.3`, `1.0.0-alpha`）は受け付けない。
- bump レベル指定 (`major` / `minor` / `patch`) や自動判定の場合は常に `X.Y.Z` 形式が生成されるため、この制約に抵触しない。

## バージョン自動判定ルール

引数が空の場合、以下の手順で次バージョンを決定する:

1. `CMakeLists.txt` から現在のバージョン `CURRENT` を読み取る。
2. `git log` で前回タグ (`v{CURRENT}`) から HEAD までのコミットメッセージと diff の概要を確認する。
3. 以下の基準で bump レベルを判定する:

| 判定 | 条件（いずれかに該当） | 例 |
|------|----------------------|-----|
| **major** | 後方互換性のない変更、プロトコル/API の破壊的変更、設定ファイル形式の非互換変更 | WebSocket メッセージ形式の変更 |
| **minor** | 新機能の追加、UI の新規画面・項目、新しい設定項目 | Opus 対応追加、新しい計測モード |
| **patch** | バグ修正、パフォーマンス改善、リファクタリング、ドキュメント修正、ビルド修正 | クラッシュ修正、CSS 調整 |

4. 判定結果とバージョン番号をユーザに提示し、確認を取る。ユーザが別のレベルやバージョンを指定したらそちらに従う。

## 手順

### 1. 事前確認

- 現在のブランチが `main` であることを確認する。`main` でなければユーザに警告して確認を取る。
- ワーキングツリーがクリーンであることを確認する（未コミットの変更がないこと）。汚れていたら中断する。
- 同じタグ (`v{VERSION}`) が既に存在しないことを確認する。

### 2. 変更内容の把握とバージョン決定

- `git log` で前回のタグから HEAD までのコミットを確認する（タグがない場合は全履歴）。
- 引数がなければ「バージョン自動判定ルール」に従って次バージョンを決定する。
- 変更内容をユーザ目線で要約し、**決定したバージョン番号と CHANGELOG に書く内容** をユーザに提示して確認を取る。

### 3. ファイル更新

以下のファイルのバージョン番号を更新する:

| ファイル | 更新箇所 |
|---------|---------|
| `CMakeLists.txt` | `project(obs-delay-stream VERSION X.Y.Z)` |
| `README.md` | 1行目のタイトル `# obs-delay-stream  vX.Y.Z` |
| `README-en.md` | 1行目のタイトル `# obs-delay-stream  vX.Y.Z` |
| `CHANGELOG.md` | 先頭に新バージョンのセクションを追加（日本語、既存フォーマットに合わせる） |

**CHANGELOG のフォーマット:**
```markdown
## vX.Y.Z — YYYY-MM-DD

### セクション名（変更内容に応じて選択）

- **機能名** — 説明
```

セクション名は変更内容に応じて `新機能`、`改善`、`修正`、`ビルド・配布`、`破壊的変更` などから適切に選ぶ。

### 4. コミット

```
git add CMakeLists.txt README.md README-en.md CHANGELOG.md
git commit -m "Release vX.Y.Z"
```

### 5. タグ作成

```
git tag vX.Y.Z
```

### 6. プッシュ

コミットとタグを両方プッシュする:

```
git push origin HEAD
git push origin vX.Y.Z
```

## 注意事項

- バージョン番号の正式な定義は `CMakeLists.txt` の `project(VERSION)` のみ。C++ コードや HTML はビルド時にマクロ / configure_file で自動展開されるため手動更新不要。
- `build.bat` のバナー表示バージョンは更新対象外。
- ファイルヘッダコメント (`.hpp` / `.cpp`) にバージョン番号は書かない。
